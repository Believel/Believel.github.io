管理输出(动态添加bundle向html文件中)

预先准备

  1. webpack.config.js文件更改输入和输出分别是多个

设定HtmlWebpackPlugin

  1. 安装npm install --save-dev html-webpack-plugin

  2. 调整webpack.config.js文件

  3. HtmlWebpackPlugin会默认生成 index.html 文件,替换之前存在的index.html

清理/dist文件夹

  1. 通常每次构建之前会清理/dist文件，安装npm install clean-webpack-plugin --save-dev
  2. 调整webpack.config.js文件

Manifest

  1. 安装npm install --save-dev webpack-manifest-plugin

使用webpack-dev-middleware

  1. 介绍：是一个中间件容器(wrapper)，它将通过 webpack 处理后的文件发布到一个服务器(server)。在内部 webpack-dev-server 它使用，然而，它可以作为一个单独的包来提供。webpack-dev-middleware 配合 express server 的示例
  2. 安装：npm install --save-dev express webpack-dev-middleware

  3. 调整webpack.config.js文件

  4. 设置自定义的express服务，创建一个server.js文件

  5. 在package.json文件中添加一个npm script，方便运行服务

  6. 在终端中执行命令npm run server,展示如下信息：现在可以通过浏览器输入(http://localhost:3000/)访问项目地址

模块热替换(Hot Module Relacement)(只适用在开发环境中)

1.作用：它允许在运行时更新各种模块，而无需进行完全刷新。

启用HMR

  1. 更新本地服务的配置，如果仅适用了webpack-dev-server

    • 调整webpack.config.js,删除print.js的入口文件，因为它被index.js使用

    • 或者通过命令更改webpack-dev-server --hotOnly

  2. 如果使用了webpack-dev-middleware,请看https://github.com/glenjamin/webpack-hot-middleware

Tree Sharking

精简输出

  1. 添加一个能够删除未引用代码(dead code)的压缩工具(minifier) - UglifyJSPlugin
  2. 安装npm i --save-dev uglifyjs-webpack-plugin
  3. 调整webpack.config.js文件内容

结论

  1. 使用 ES2015 模块语法（即 import 和 export
  2. 引入一个能够删除未引用代码(dead code)的压缩工具(minifier)（例如 UglifyJSPlugin）

生产环境构建(product)

配置

  1. 安装’npm install –save-dev webpack-merge’,并将之前指南中已经成型的那些代码再次进行分离
  2. 删除webpack.config.js,创建webpack.common.js, webpack.dev.js, webpack.prod.js等文件

  3. 重新配置npm scripts

  4. 在生产环境中使用source-map,在开发环境中使用inline-source-map

指定环境

  1. 许多 library 将通过与 process.env.NODE_ENV 环境变量关联，以决定 library 中应该引用哪些内容
  2. 在生产环境中，调整webpack.prod.js文件
